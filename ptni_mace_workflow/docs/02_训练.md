# 02 训练

训练模块把每一次 MACE 训练限制在独立 run 目录中，避免不同模型共享旧的 `logs/`、`checkpoints/`、`results/`。

## 入口脚本

| 脚本 | 功能 |
| --- | --- |
| `train_mace_ptni_ft.sh` | 基于 `medium-mpa-0` fine-tune |
| `train_mace_ptni_scratch.sh` | 使用相同数据和主要超参数从头训练 |
| `export_best_model_from_run.sh` | 从某个训练 run 导出 best-loss `.model` 到 `mace_workspace/models/<tag>/model.model` |
| `check_mace_training_progress.py` | 解析训练 log，查看 epoch、loss 和 RMSE |
| `check_mace_training_progress.sh` | shell 包装入口 |
| `watch_mace_training_progress.sh` | 周期性监视训练进度 |
| `inspect_mace_run_settings.py` | 提取训练参数和模型设置 |
| `export_mace_checkpoint_to_model.py` | 手动把指定 checkpoint 转为 `.model` |

## Fine-tune

```bash
MACE_WORKSPACE=mace_workspace \
SAVE_ALL_CHECKPOINTS=1 WANDB=1 \
WANDB_PROJECT=ptni-mace \
WANDB_NAME=ptni_binary_mace_ft \
bash ptni_mace_workflow/training/train_mace_ptni_ft.sh \
  --dataset ptni_split \
  --run-name ptni_binary_mace_ft
```

输出目录：

```text
mace_workspace/runs/training/ptni_binary_mace_ft/
  checkpoints/
  logs/
  results/
  models/
  wandb/
  run_manifest.json
```

## Scratch

```bash
MACE_WORKSPACE=mace_workspace \
SAVE_ALL_CHECKPOINTS=1 WANDB=1 \
WANDB_PROJECT=ptni-mace \
WANDB_NAME=ptni_binary_mace_scratch \
bash ptni_mace_workflow/training/train_mace_ptni_scratch.sh \
  --dataset ptni_split \
  --run-name ptni_binary_mace_scratch
```

scratch 默认沿用当前基准超参数：

| 参数 | 默认值 |
| --- | --- |
| `r_max` | `6.0` |
| `num_interactions` | `2` |
| `correlation` | `3` |
| `hidden_irreps` | `128x0e + 128x1o` |
| `batch_size` | `4` |
| `energy_weight` | `1.0` |
| `forces_weight` | `10.0` |
| `lr` | `0.01` |
| `weight_decay` | `5e-7` |

## 续算

```bash
MACE_WORKSPACE=mace_workspace \
RESTART_LATEST=1 SAVE_ALL_CHECKPOINTS=1 WANDB=1 \
bash ptni_mace_workflow/training/train_mace_ptni_ft.sh \
  --dataset ptni_split \
  --run-name ptni_binary_mace_ft
```

如果同名 run 已有 checkpoint 且没有设置 `RESTART_LATEST=1`，脚本会拒绝继续，避免误从 foundation model 重新开始并混入同一目录。

## 导出 best-loss model

```bash
bash ptni_mace_workflow/training/export_best_model_from_run.sh \
  --workspace mace_workspace \
  --run-name ptni_binary_mace_ft \
  --model-tag ft_best_loss \
  --metric loss
```

输出：

```text
mace_workspace/models/ft_best_loss/model.model
mace_workspace/models/ft_best_loss/model_manifest.json
```

## 运行约束

- 训练入口会进入 `mace_workspace/runs/training/<run_name>/` 再调用 `mace_run_train`。
- MACE 的 `checkpoints_dir`、`log_dir`、`results_dir`、`model_dir` 均相对 run 目录。
- 以后不要把新训练直接写入仓库根目录的 `logs/`、`checkpoints/`、`results/`。
