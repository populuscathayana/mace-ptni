# 04 外推任务验证

外推任务验证模块用于检查模型在训练集以外场景的物理可信度。重点不是只看一个 RMSE，而是看模型能否复现合理的能量面、势垒和优化行为。

## 统一入口

```bash
bash ptni_mace_workflow/benchmarks/run_benchmark_suite.sh \
  --workspace mace_workspace \
  --model-tag ft_best_loss \
  --suite lattice,strained_neb,pt111_pes,distance_scan,np_singlepoint,np_relax_neb \
  --device cuda
```

CPU smoke test：

```bash
bash ptni_mace_workflow/benchmarks/run_benchmark_suite.sh \
  --workspace mace_workspace \
  --model-tag ft_best_loss \
  --suite lattice,strained_neb,pt111_pes,distance_scan,np_singlepoint,np_relax_neb \
  --device cpu \
  --smoke
```

## Benchmark 简表

| suite 名称 | 输出目录 | 主要判断 |
| --- | --- | --- |
| `lattice` | `mace_workspace/runs/benchmarks/lattice/<model_tag>/` | Pt/Ni fcc 平衡晶格常数和能量曲线 |
| `strained_neb` | `mace_workspace/runs/benchmarks/strained_neb/<model_tag>/` | 不同应变下初态、过渡态、终态和活化能 |
| `pt111_pes` | `mace_workspace/runs/benchmarks/pt111_pes/<model_tag>/` | Pt111 表面 Pt adatom 势能面、吸附能相对值、MACE-DFT 差值 |
| `distance_scan` | `mace_workspace/runs/benchmarks/distance_scan/<model_tag>/` | PtNi slab 顶部单个可动原子沿 Direct-z/第三晶格矢量方向拉远时的单点能稳定性 |
| `np_singlepoint` | `mace_workspace/runs/benchmarks/np_singlepoint/<model_tag>/` | NP NEB 00/01/02 单点能和势垒 |
| `np_relax_neb` | `mace_workspace/runs/benchmarks/np_relax_neb/<model_tag>/` | 优化初末态后执行 MLIP CI-NEB，并审计异常几何 |

## 晶格常数

默认扫描范围：

| 元素 | 范围 |
| --- | --- |
| Pt | `3.75:4.15:0.01` |
| Ni | `3.35:3.75:0.01` |

Ni 的范围已经固定为 `3.35-3.75 A`，避免旧脚本里误用 Pt 范围。

## Pt111 PES

输入默认来自：

```text
mace_workspace/inputs/pt111/
  hex_point_000/POSCAR
  hex_point_001/POSCAR
  ...
  hex_point_origin/
```

该任务固定 slab 中除最后一个 adatom 外的原子，adatom 固定 x/y，只优化 z。若存在 `hex_point_origin`，会额外计算以无 adatom 结构为参照的相对能量，并重新绘制共享色标的图。

## PtNi slab 距离稳定性

输入默认来自：

```text
mace_workspace/inputs/PtNi-diffusion/
  111/POSCAR
  100/POSCAR
```

这两个 POSCAR 使用 selective dynamics 标记，脚本默认寻找唯一带 `T` 的原子，例如 `F F T`，将其视为可动原子。任务流程：

1. 沿第三晶格矢量加厚真空层，默认增加 `20 A`，并保持 slab 笛卡尔坐标不变，避免把 slab 拉伸。
2. 沿第三晶格矢量方向逐步增加可动原子的 Direct-z 坐标，默认距离扫描为 `0:5:0.05 A`。
3. 对每个距离做 MACE 单点能，输出相对于第一个点的能量。
4. 同时输出两类横坐标：绝对位移 `delta_A`，以及相对最近下层层间距的 `delta_over_layer_spacing`。

单独运行：

```bash
bash ptni_mace_workflow/benchmarks/run_benchmark_suite.sh \
  --workspace mace_workspace \
  --model-tag ft_best_loss \
  --suite distance_scan \
  --device cuda
```

常用环境变量：

| 变量 | 默认值 | 含义 |
| --- | --- | --- |
| `DISTANCE_SCAN` | `0:5:0.05` | 沿第三晶格矢量方向的绝对位移扫描，单位 A |
| `DISTANCE_SCAN_VACUUM` | `20` | 额外真空厚度，单位 A |
| `DISTANCE_SCAN_SURFACES` | `111 100` | 要扫描的子目录 |
| `DISTANCE_SCAN_INPUT_DIR` | `mace_workspace/inputs/PtNi-diffusion` | POSCAR 输入根目录 |
| `DISTANCE_SCAN_WRITE_STRUCTURES` | `0` | 是否写出每个距离点的 POSCAR |

输出包括：

- `*_scan.csv`：每个距离点的能量、相对能、轴向力和最大力。
- `*_energy_vs_delta_A.png`：横坐标为绝对位移。
- `*_energy_vs_layer_spacing.png`：横坐标为相对层间距。
- `*_mobile_atom_force.png`：可动原子的轴向力和力范数。
- `*_report.html`：可直接打开的本地 HTML 汇总。

这个 benchmark 用于检查模型在原子逐渐远离表面时是否出现非物理振荡、突然能量下降或力尖峰。它不是 NEB，也不做结构优化。

## NP 外推

NP benchmark package 默认来自：

```text
mace_workspace/datasets/NP_benchmark_package/np_neb_benchmark_all.extxyz
```

`np_singlepoint` 只做 00/01/02 单点预测和势垒比较。

`np_relax_neb` 会优化端点并执行 CI-NEB，同时保留以下诊断：

- 优化后 IS/TS/FS 与原 00/01/02 的 RMSD。
- 最大单原子 RMSD。
- detached atom / collision / force warning。
- NEB barrier 和原始单点 barrier 的差异。

## semi-rfKMC / MCMD 动力学雏形

vacancy-mediated semi-rfKMC / MCMD 不属于固定 benchmark suite，而是独立运行入口：

```bash
python -m ptni_mace_workflow.mcmd.run_vacancy_mcmd \
  --workspace mace_workspace \
  --input mace_workspace/inputs/mcmd/POSCAR \
  --model-tag ft_best_loss \
  --run-name np_vacancy_mcmd_test \
  --vacancy-site-index 0 \
  --temperature 800 \
  --mc-steps 20 \
  --md-steps 200 \
  --neb-images 5 \
  --neb-fmax 0.05 \
  --device cuda
```

该流程用 close-packed site reconstruction 提供 He/vacancy 候选位点，用 ASE+MACE 做可选 MD。默认 `atom-random` semi-rfKMC 会先随机选一个低配位原子，再对该原子周围所有合法 He/vacancy 位点都执行显式 CI-NEB，并按 `nu * exp(-Ea/kBT)` 的局部速率概率选择事件。它不是完整枚举全体系事件目录的 rfKMC；详细说明见 `08_MCMD框架.md`。

## 成功判据

- 每个 benchmark 都有独立的 `<model_tag>` 输出目录。
- `benchmark_suite_manifest.csv` 记录模型、任务和输出目录。
- NP relax+NEB 的结果不能只看 barrier，必须同时检查几何审计表。
- 高能异常点优先判断是否发生非物理路径漂移，而不是直接解释为势垒误差。
